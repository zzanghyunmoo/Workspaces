# ZZA-70 CE 3.19.0 upstream lock and inventory

## 현재 기능 상태

PR [#21](https://github.com/zzanghyunmoo/oh-my-harness/pull/21)이 squash merge되어
Compound Engineering 3.19.0의 immutable upstream identity와 29개 direct skill inventory가
`oh-my-harness`의 U1 trust root로 제공된다. Merge commit은
`143c2e827245c46f4ec367567126f1b255ac7e00`이다.

## 주요 동작과 경계

- canonical source는 `EveryInc/compound-engineering-plugin`의
  `compound-engineering-v3.19.0` tag, commit
  `1756c0b9f3cf94493f287ea29ae766ad668fb7cf`, tree
  `808d20cc08a2b45e0200e68f5b9f604c55cf8a06`으로 고정된다.
- lock은 manifest, `bun.lock`, signed payload receipt, executable set, package scripts와
  inventory digest를 함께 고정한다.
- inventory는 `skills/*/SKILL.md`의 29개 direct skill을 Git blob ID로 기록하고 `lfg`를
  포함한다. Upstream skill body는 복사하지 않는다.
- verifier는 pinned Git objects만 읽으며 symbolic tag, replacement refs, alternates,
  promisor repository, corrupt object database, npm-modified `PATH` shim과 credential-bearing
  artifact를 거부한다.
- offline verification은 content identity와 configured origin을 증명하지만 GitHub에서
  authenticated acquisition됐다고 주장하지 않는다.
- U1은 runtime adapter, U2/U3 schema, U16 feasibility proof와 116-cell conformance를
  구현하지 않는다.

## 검증 결과

- `npm run test:harness`: 15개 canonicalization, identity, hostile Git/target,
  no-mutation 회귀 테스트가 통과했다.
- canonical CE checkout에서 explicit generation 후 verification과
  `git diff --exit-code -- harness/`가 byte-identical no-op으로 통과했다.
- `npm run profile:verify`: 4개 profile과 profile lock이 deterministic·secret-free다.
- `npm run test:workspace-connectors`: 기존 connector/setup/OMP 31개 테스트가 통과했다.
- 최신 head `0bb4163ae802c66b04334d946a5aa05c96a4f9f0`의 code/doc review와 workflow
  pre-merge gate가 통과했다.
- Repository에 GitHub Actions workflow가 없어 remote check-run은 생성되지 않았다.

## 운영 및 사용 시 주의사항

- 검증에는 pinned commit의 모든 object가 materialized된 canonical checkout이 필요하다.
  Partial/promisor checkout은 fail-closed로 거부한다.
- Git executable은 ambient `PATH`에서 찾지 않는다. Windows나 비표준 설치에서는
  `OH_MY_HARNESS_GIT_EXECUTABLE`에 repository 밖의 trusted absolute path를 지정한다.
- `harness:upstream:verify`는 read-only다. Artifact 갱신은 `generate --write`를 명시해야
  하며 inventory가 먼저, 이를 bind하는 lock이 나중에 publish된다.
- Package scripts, plugin code와 skill body는 inventory derivation 중 실행하지 않는다.
- U2-U18이 남아 있으므로 ZZA-70은 `In Review`와 completion pending을 유지한다.
